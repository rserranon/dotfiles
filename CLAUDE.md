### Stow Layout Rule

Every stow package uses **double-nesting**: `<pkg>/<pkg>/` ‚Üí `~/.config/<pkg>/`.
Example: `nvim/nvim/init.lua` ‚Üí `~/.config/nvim/init.lua`

**Packages managed by `install.sh` stow loop:**
`alacritty  aliases  git  homebrew  nvim  starship  tmux  zsh`

Note: `claude`, `copilot`, `osx` are **not** in the stow loop ‚Äî handled with explicit `ln -sf` calls.

**Special symlinks** (outside `~/.config`):

| Source | Target | Notes |

### Code Patterns

- **Lua plugins**: return a single table; use `opts =` (not `config =`) when possible; declare keybinds in `keys =`
- **Shell scripts**: `set -euo pipefail`; colored output via `pass()`/`fail()`/`warn()` functions
- **Aliases**: grouped by domain with `# comment` headers; no spaces around `=`
- **Stow packages**: double-nested (`pkg/pkg/file`); never put files directly at package root
- **JSON**: ignored globally (`*.json`) with `!claude/**/*.json` exception for Claude configs
- **Secrets**: always guarded with `[[ -f path ]] && source path`; never committed
